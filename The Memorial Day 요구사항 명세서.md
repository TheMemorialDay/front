### 로그인 화면
  1. 사용자는 아이디와 비밀번호를 입력하여 로그인을 할 수 있음.

  4. '비밀번호 찾기' 클릭하여 비밀번호 찾기 화면으로 바뀐다.

  8. SNS 로그인으로 카카오, 네이버, 구글 로그인을 할 수 있음.
  9. 만약 가입된 SNS 회원이라면 STORES으로 이동, 만약 가입되지 않은 SNS 회원이라면 회원가입 화면으로 이동
  10. '회원가입' 클릭하여 회원가입 페이지로 이동한다.

 2. 가게 검색박스 밑에 존재하는 케이크 카테고리 아이콘 들은, 하나만 선택 가능하도록 한다. 선택시에 해당 카테고리가 포함된 가게가 나열된다. 클릭 시에 해당 아이콘이 강조되는 효과를 준다.
 3. 필터링(주소(구,동), 태그(테마, 복수선택가능), 당일케이크, 픽업가능요일)이 존재하며, 선택시에 필터링 된 가게들이 나열된다.
 4. 한 페이지에 가게 8개까지(바둑식 배열) 나오게 하고 페이지네이션을 이용한다.
 5. 필터 박스 아래에 사용자가 선택한 필터 종류를 보이게 한다.
 6. 선택한 필터 들은 개별 X 버튼이 존재하며 버튼을 클릭 시 삭제가 된다.
 7. 초기화 버튼을 클릭하면 선택한 필터들이 모두 초기화 된다.
 8. 정렬 방식(디폴트: 무작위), 사용자 지정 정렬 방식 : 리뷰 수, 별점 높은 수, 찜 수가 있으며, 클릭 시 정렬된다. 
 9. 가게박스에는 (가게 이미지, 가게이름, 주소, 별점, 리뷰개수, 찜개수)가 포함된다.
 10. 가게를 클릭하면 해당 가게 페이지로 이동한다.
 11. 가게 옆에 하트 버튼을 누르면 찜하기 기능이 적용되고 빨간 하트로 바뀐다. 다시 누르면 취소(빈하트)된다.
 12. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.
 

 ### STORES 옵션
 1. 테마 - (심플, 화려함, 펑키, 크리스마스, 아이돌, 졸업, 귀여움, 러블리, 재미, 할로윈, 신년, 효도, 연인, 어린이, 웨딩, 취업/승진)이 있다. 복수 선택 가능, 모달 창의 체크 박스로 구현한다.
 2. 픽업 가능 요일 - 일~월 복수 선택 가능, 모달 창의 체크박스로 구현한다.
 3. 당일 케이크 여부 - 클릭 이벤트 처리, 클릭되면 검은 배경이고 당일 케이크 가능한 가게만 띄워준다. 클릭 해제시 흰 배경으로 돌아온다.  
 4. 주소 - 구/군 선택을 먼저 하도록 한다. 그 뒤에 읍/동을 선택하도록 한다. 구/군을 지우면 읍/동도 없어지도록 한다.  
 5. 필터링 버튼에 있는 내용들은 가나다 순으로 정렬된다.
 6. 필터 하나씩 삭제될 때 마다 가게 리스트가 바뀐다.
 7. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


 ## 가게 페이지
 1. 가게 썸네일과 가게 정보(이름, 별점, 주소, 영업시간, 짧은 설명)를 상단에 위치시킨다. 가게 정보 드래그만 가능하도록 한다.
 2. 가게 페이지 중앙바에 주문하기, 매장정보, 문의하기, 리뷰 카테고리를 위치시킨다.
 3. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


 ### 가게 페이지 - 주문하기
 1. 페이징 처리 없이 모든 상품을 띄워준다.
 2. 상품의 순서는 등록한 순서대로 띄워준다.
 3. 상품을 클릭하면 케이크 주문서 작성 페이지로 이동한다.
 4. 케이크 위에 마우스를 올리면(hover) 배경을 약간 하얗게 하고 케이크 이름, 짧은 설명, 가격을 보이게 한다.
 5. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


 ### 케이크 주문서 작성 페이지
 1. 케이크 사진을 여러장 보여줄 수 있도록 한다. 좌우 화살표로 사진을 넘길 수 있도록 한다.
 2. 케이크 이름, 설명, 기본 가격, 픽업 일시(yyyy년 mm월 dd일 hh시 mm분), 옵션(라디오 버튼), 유의사항 확인, 요청사항, 수량(default : 1)을 선택할 수 있다.
 3. 옵션(라디오 버튼), 유의사항 확인 버튼은 필수 사항이다.
 4. 유의 사항 아래 오른쪽에 총 금액을 띄워준다.
 5. 하단에 장바구니 버튼이 존재하며, 클릭 시 window.confirm을 뛰우며 확인을 누르면 장바구니로 이동, 취소를 누르면 해당 페이지에 머문다.  
 6. 하단에 주문하기 버튼이 존재하며, 주문하기 버튼을 누르면 '주문서를 제출할까요?' window.confirm을 띄운다.
 7. 확인을 누르면 주문 요청 완료 페이지로 이동하며 주문서가 제출되고, 아니요를 누르면 해당 페이지에 머물며 주문서는 제출되지 않는다.
 8. 인증되지 않은 유저가 주문하기 버튼을 눌렀을 때 alert 창을 띄워 '로그인 후 이용해주세요.'를 띄운다.
 9. 필수 사항에서 하나라도 선택하지 않았을 때 alert 창을 뛰어 '필수 사항을 선택해주세요.'를 띄운다.
 10. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

 
 ### 주문 요청 완료 페이지
 1. '주문 요청이 완료 되었습니다. 사장님 확인을 기다려 주세요. 휴대폰 메시지로 알려드릴게요!'라는 문구가 화면에 표시된다.
 2. 주문 내역 확인 버튼을 누르면 주문 상세 내역페이지로 이동한다.
 3. '둘러보기' 버튼을 누르면 STORES 화면으로 이동한다.   
 4. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

 2. 사장님이 등록하신 매장을 소개하는 글을 지도 아래에 위치시킨다.
 3. 사장님이 등록하신 매장을 소개하는 사진을 아래에 위치시킨다.
 4. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.
 

 ### 가게 페이지 - 문의하기
 1. 사장님이 자신과의 연락할 수 있는 연락수단과 방법을 적어놓는다.
 2. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


 ### 가게 페이지 - 리뷰
 1. 리뷰 별로 내용, 별점, 작성일, 사진, 상품을 가지도록 한다.

 3. 사진은 썸네일 한장만 보이게 하고, 클릭했을 때 뒷 배경은 약간 어둡게 설정하고 사진을 크게 보여주고 좌우 버튼을 눌러 나머지 사진들을 옆으로 넘기며 볼 수 있게 한다.  
 4. 4개의 리뷰를 한 화면에 보이게 하고, 좌우 버튼을 이용하여 화면을 넘기게 한다.
 5. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

 ## HOW 페이지
 1. 웹 사이트의 전반적인 사용방식에 대한 페이지이다.
 2. 각각의 버튼들을 호버하면 버튼은 강조 처리가 되며 클릭 시, 해당 버튼에 대한 설명을 박스를 늘려 표기한다.
 3. 설명 박스 안에는 사이트의 캡처본을 통하여 사용자의 전반적인 이해를 돕는다.
 4. 백엔드 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.
 
 ## SUPPORT 페이지
 1. Notice 와 Q&A 가 존재한다.
 2. Notice를 클릭 시 Notice 테이블로 렌더링 된다.
 3. Q&A를 클릭 시 Q&A 테이블로 렌더링 된다.
 4. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

 ### Notice
 1. 게시글, 제목, 작성일이 존재한다.
 2. 검색창에서 제목을 이용하여 검색을 한다.
 3. 한 페이지에 10개 씩 보이도록 하며, 초과되는 페이지는 페이지네이션을 이용한다.
 4. 하나의 게시글을 클릭하면 해당 게시글의 상세페이지로 이동한다.
 5. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


 ### Q&A
 1. 게시글, 제목, 작성자, 작성일, 상태가 존재한다.
 2. 검색창에서 제목 및 작성자를 이용하여 검색을 한다.
 3. 한 페이지에 10개 씩 보이도록 하며, 초과되는 페이지는 페이지네이션을 이용한다.
 4. 하나의 게시글을 클릭하면 해당 게시글의 Q&A 상세페이지로 이동한다.
 5. 우측 상단 '등록' 버튼을 누르면 질문 작성 페이지로 이동한다.
 6. 로그인되지 않은 유저가 Q&A에 들어왔을 때 등록 버튼이 존재하지 않는다.
 7. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.
 

 ### Q&A 작성 페이지
 1. 제목, 본문을 작성 할 수 있다. 본문의 최소 글자는 공백포함 15자로 제한한다.
 2. 오른쪽 상단에 위치한 등록 버튼을 클릭하면, 게시글이 등록이 되며 해당 게시글의 Q&A 상세페이지로 이동한다.
 3. 오른쪽 상단에 위치한 취소 버튼을 클릭하면, Q&A 리스트페이지로 이동한다.
 4. 본문의 최소글자를 만족시키지 못하였을 때 alert를 뛰워 '최소 글자를 채워주세요.' 라고 한다. 
 5. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

 ### Q&A 상세 페이지
 1. 해당 게시글의 제목과 본문, 작성일이 보인다.
 2. 오른쪽 상단에 위치한 수정과 삭제 버튼은 해당 게시글의 작성자만 보이게 한다.
 3. 오른쪽 상단에 위치한 수정 버튼을 클릭하면 해당 게시글의 Q&A 작성 페이지로 이동한다.
 4. 수정버튼을 통해 들어간 해당 게시글의 Q&A 작성 페이지에는 기존의 작성되어 있던 내용들이 보여지며, 오른쪽 상단의 수정 버튼을 클릭하면, 수정이 완료되며 해당 게시글의 Q&A 상세 페이지로 이동한다.  
 5. 오른쪽 상단에 위치한 삭제 버튼을 클릭하면 window.confirm 창을 띄워 확인을 클릭시 삭제, 아니요를 클릭시 아무 동작을 수행 안한다.  
 6. 관리자는 답변이 되지 않은 질문에 한하여 답변을 입력할 수 있다.
 7. 관리자의 답변의 입력이 완료되면 하단에 본문 밑에 회색 답변 박스가 생성되어 관리자의 답변이 회원에게 보여진다.
 8. 수정된 내용의 본문이 최소글자를 만족시키지 못하였을 때 alert를 뛰워 '최소 글자를 채워주세요.' 라고 한다. 
 9. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

## join 페이지 (사장 권한 요청 페이지) - **수정 덜 됨
1. 사업자 등록 번호, 사업자 등록증 파일을 등록한다.
2. 하단에 위치한 등록 버튼을 누르면 가게 등록 요청이 완료되었음을 알리는 페이지로 이동한다. (** 사장 등급 변경 안내 관련으로 변경해야 함)
3. 하단에 위치한 취소 버튼을 누르면 메인 화면으로 이동한다.
4. 사업자등록번호 인증을 수행하기 전에 숫자 10자 형식인지 확인하여 숫자 10자 형식이 아니라면 사업자등록번호 입력칸 아래쪽에 '숫자 10자 입력해주세요.'라는 빨간색 에러 문구를 표시함
5. 사업자 등록증 파일과 가게 대표이미지를 첨부파일로 입력 받는다.
6. 사업자 등록 번호, 사업자 등록증 파일 중 하나라도 입력하지 않으면 등록버튼이 활성화 되지 않는다.
7. 사업자 등록 번호, 사업자 등록증 파일 중 하나라도 입력하지 않은 상태에서 등록버튼을 누르면 alert를 띄워 모든 입력을 하도록 한다. 문구는 '정보를 모두 입력해주세요.'.
8. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.\

## 가게 등록 페이지 - **수정 덜 됨
1. 가게명, 가게 연락처, 가게 대표 사진(default = '임의의 사진 (ex: 준비중)')을 입력받는다.
2. 가게명은 별다른 제약조건없이 입력 받는다.
3. 가게주소는 Daum 주소 API를 이용하여 주소를 입력 받는다.
4. 가게 연락처는 별다른 제약조건없이 입력 받는다.


### 가게 등록 요청 완료 페이지
1. '가게 등록 요청이 완료 되었습니다. 관리자 확인을 기다려 주세요. 휴대폰 메시지로 알려드릴게요!' 라는 문구가 화면에 표시된다.
2. '확인' 버튼을 누르면 메인페이지로 이동한다. 
3. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

## 마이페이지
1. 사용자의 페이지와 사장님의 페이지로 나뉜다.
2. 사용자의 페이지에는 개인정보 관리, 리뷰 관리, 주문내역, 찜한 가게가 존재한다.
3. 사장님의 페이지에는 기존의 사용자의 페이지에서 가게 관리, 상품 관리, 주문 관리, 매출 관리가 추가되어 존재한다.
4. 백엔드 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

### 개인정보 관리
1. 비밀번호 입력을 받아 본인인증을 수행하여 관리페이지로 들어간다.
2. 관리페이지로 들어오면 기존에 입력해 놨던 개인정보들을 불러온다.
3. 수정버튼을 클릭하여 수정한 개인정보를 저장할 수 있다.
4. 수정버튼을 클릭하여 수정할 수 있는 정보는 이름, 비밀번호, 생년월일, 성별, 전화번호가 있다.
5. 비밀번호 칸 옆의 비밀번호 재설정 버튼을 눌러 비밀번호 재설정 모달창을 이용하여 재설정을 수행한다.
6. 비밀번호 모달창에서 완료 버튼을 누르면 창이 닫히며 비밀번호가 변경된다. 취소를 누르면 모달창이 닫히며 아무 동작하지 않는다.
7. 전화번호의 변경은 해당 전화번호의 인증번호를 받아 진행한다.
8. 전화번호 인증을 수행하기 전에 숫자 11자 형식인지 확인하여 숫자 11자 형식이 아니라면 전화번호 입력칸 아래쪽에 '숫자 11자 입력해주세요.'라는 빨간색 에러 문구를 표시함  
9. 전화번호 인증의 경우 중복된 전화번호라면 전화번호 입력칸 아래쪽에 '중복된 전화번호입니다.'라는 빨간색 에러 문구를 표시하고 정상적으로 전화번호 인증 문자가 전송되었다면 전화번호 입력칸 아래쪽에 '인증번호가 전송되었습니다.'라는 초록색 알림 문구가 표시됨  
10. 인증번호 입력칸은 인증 문자가 정상적으로 전송되었을 때 화면에 표시함
11. 인증번호는 인증 확인 버튼을 클릭하여 해당하는 전화번호의 인증번호를 확인함
12. 인증번호를 입력했을 때만 인증 확인 버튼을 활성화
13. 인증번호를 잘못 입력했을 때는 인증번호 입력칸 아래쪽에 '인증번호가 일치하지 않습니다.'라는 빨간색 에러 문구를 표시하고 정상적으로 입력했을 때는 입력칸 아래쪽에 '인증번호가 확인되었습니다.'라는 초록색 알림 문구를 표시함  
14. 탈퇴 버튼을 클릭하면 window.confirm을 이용하여 탈퇴 동의를 확인 후 확인 버튼을 클릭하면 탈퇴, 아니요를 누르면 아무 동작하지 않는다.  
15. 이름, 비밀번호, 생년월일, 성별, 전화번호, 전화번호 인증 중 하나라도 비워져 있으면 수정버튼이 활성화 되지 않는다.
16. 이름, 비밀번호, 생년월일, 성별, 전화번호, 전화번호 인증 중 하나라도 입력하지 않고 수정버튼을 누르면 alert를 띄워 모든 입력을 하도록 한다. 문구는 '정보를 모두 입력해주세요.'.  
17. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


### 주문내역
1. 각 주문 별 왼쪽상단에 5단계로 상태가 구분뵌다. (주문 승인 대기, 주문취소, 결제대기, 결제완료, 완료)
2. 승인대기중 상태는 사장님의 주문 승인을 기다리는 상태이다. 또한 유저가 주문취소 버튼을 클릭하여 주문을 취소할 수 있다.
3. 주문취소는 픽업일로 부터 3일전에는 제약없이 할 수 있고, 픽업일로 부터 3일이 남지 않았을 경우 주문취소 사유를 작성하여야 한다.
4. 주문취소 사유는 라디오버튼으로 제출할 수도 있고 기타버튼을 클릭하여 사유를 작성하여 제출 할 수도 있다. (기타 사유 default : '기타')
5. 주문취소 사유 제출이 완료 되었을 때 주문의 상태는 주문 취소로 변경되며 버튼들이 보이지 않는다.
6. 사장님이 주문을 수락하면 주문 승인 대기에서 결제 대기중 상태로 변경되며 우측의 버튼은 결제와 주문취소로 변경된다.
7. 결제 버튼을 클릭하면 결제창으로 이동한다.
8. 결제가 완료되면 결제대기중에서 결제 완료 상태로 변경되며 우측의 버튼이 사라진다.
9. 고객의 픽업이 완료되고 사장님이 완료처리를 한다면 결제완료에서 완료 상태로 변경되며 우측의 버튼에 리뷰쓰기로 변경된다.
10. 리뷰쓰기 버튼을 누르면 리뷰 작성 페이지로 이동한다.
11. 우측 상단에는 각 주문별 주문 코드가 존재하며 10자리의 랜덤한 숫자로 구성된다.
12. 각 주문 내역 박스에는 상품사진, 가게명, 상품명, 상품 옵션(사이즈, 맛, 요청사항), 총 금액, 픽업 일시가 존재한다.
13. 가게명을 눌렀을때 가게 페이지로 이동한다.
14. 주문 일자별로 각 날짜에 해당하는 주문 내역 박스가 존재한다.
15. 주문취소 사유를 선택하지 않고 제출을 클릭하였을 때 alert를 뛰어 '주문취소 사유를 선택해주세요.!' 라고 한다.
16. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


### 리뷰 관리
1. 본인이 작성했던 리뷰리스트들을 가게명을 상단에 띄어주고 가게별로 최근 주문순으로 나열한다.
2. 가게명을 눌렀을때 가게 페이지로 이동한다.

5. 사진은 썸네일 한장만 보이게 하고, 클릭했을 때 뒷 배경은 약간 어둡게 설정하고 사진을 크게 보여주고 좌우 버튼을 눌러 나머지 사진들을 옆으로 넘기며 볼 수 있게 한다.  
6. 4개의 리뷰를 한 화면에 보이게 하고, 좌우 버튼을 이용하여 화면을 넘기게 한다.
7. 리뷰 별로 삭제 버튼을 클릭하면 window.confirm 창을 이용하여 확인 버튼을 누르면 리뷰가 삭제되고 아니요를 누르면 아무 동작도 하지 않는다.
8. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

### 가게 관리 - 가게 관리
1. 가게 등록에서 입력했던 가게명, 가게 연락처, 가게 주소, 대표이미지 (default = '임의의 사진 (ex: 준비중)')을 수정 가능 상태로 만든다.
2. 사업자 등록 번호와 사업자 등록증은 따로 관리자에게 문의를 통해 변경하도록 비활성화 해놓는다.
3. 태그, 가게 소개글, 픽업 가능 요일, 매장정보, 문의하기 수정할 수 있도록 한다.
4. 태그 선택 방식은 태그 버튼을 클릭하면 검은색으로 변경되며, 태그는 최대 5개까지 선택 가능하다.
5. 수정버튼을 누르면 수정이 완료된다.
6. 가게명, 가게 주소, 픽업 가능 요일 중 하나라도 비워져 있으면 수정버튼이 활성화 되지 않는다.
7. 가게명, 가게 주소, 픽업 가능 요일 중 하나라도 입력하지 않고 수정버튼을 누르면 alert를 띄워 모든 입력을 하도록 한다. 문구는 '정보를 모두 입력해주세요.'. 
8. 취소 버튼을 누르면 사장님 마이페이지로 이동한다.
9. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.

### 주문 관리
1. 날짜별로 케이크 픽업 날짜가 가까운 순으로 정렬한다.
2. 각 주문 별 왼쪽상단에 5단계로 상태가 구분된다. (승인대기중, 주문취소, 결제대기중, 결제완료, 완료)
3. 승인대기중 상태는 고객이 주문요청을 하여 사장님의 주문 승인을 기다리는 상태이다. 사장님은 승인 또는 거부를 눌러 상태를 변경할 수 있다.
4. 승인 버튼을 누를 시 상태는 결제대기중으로 변경되며 고객의 결제를 기다리는 상태이다. 우측의 승인과 거부 버튼은 비활성화 상태가 된다.
5. 또한 승인 버튼을 눌렀을 때 고객에게 결제 요청 문자를 보낸다.
6. 거부 버튼을 누를 시 사장님은 거부사유 모달창을 띄워 거부사유를 작성해야한다.
7. 거부사유는 라디오버튼으로 제출할 수도 있고 기타버튼을 클릭하여 사유를 작성하여 제출할 수도 있다. (기타 사유 default : '기타')
8. 거부 사유를 제출하였을 때 주문한 고객에게 문자로 거부사유를 보낸다,
9. 주문거부 사유 제출이 완료 되었을 때 주문의 상태는 주문 취소로 변경되며 버튼들이 보이지 않는다.
10. 고객이 결제를 하였으면 결제대기에서 결제완료로 상태가 변경되며 우측 버튼이 완료버튼으로 변경된다.
11. 고객이 픽업까지 완료하였을 때 직접 완료버튼을 눌러 완료처리를 한다. 그러면 상태가 픽업완료로 변경되며 완료 버튼이 비활성화 된다.
12. 우측 상단에는 각 주문별 주문 코드가 존재하며 10자리의 랜덤한 숫자로 구성된다. 이 주문 코드는 고객의 주문 내역과 동일하다.
13. 각 주문 관리 박스에는 상품사진, 상품명, 상품 옵션(사이즈, 맛, 요청사항), 고객정보(이름 / 연락처), 총 금액, 픽업 일시가 존재한다.
14. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


### 매출 관리 페이지 - ** 제에에에에일 나중에 시간 남으면 구현!!!
1. 주문 관리 리스트 중에서 상태가 완료인 주문만 그대로 가져온다.
2. 완료처리된 주문들을 월별로 최근 주문순으로 나타낸다.
3. 월별로 매출을 집계해서 상단에 그래프로 매출현황과 금액을 나타낸다.
4. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.


### 장바구니
1. 각 장바구니 박스에는 상품사진, 가게명, 상품명, 상품 옵션(사이즈, 맛, 요청사항), 총 금액, 픽업 일시가 존재한다.
2. 각 장바구니 박스는 최근순으로 정렬된다.
3. 체크 박스를 활용하여 장바구니 박스를 선택하여 결제를 할 수 있다.
4. 체크 박스를 선택한 상품 가격만 총 금액에 포함된다.
5. 체크 박스를 아무것도 선택하지 않고 결제하기를 클릭하였을 때 alert를 띄워 '상품을 선택해주세요.' 라고 한다.
6. 각 장바구니 박스별 우측에 삭제버튼을 클릭하면 window.confirm 이 나타나며 '상품을 삭제하시겠습니까?' 예를 누르면 상품이 삭제되고 아니요를 누르면 아무 동작도 하지 않는다.
7. 체크 박스를 선택되었을 때 결제하기를 클릭하면 결제창으로 이동된다.
8. 백엔드 서버 에러, 데이터베이스 서버 에러의 경우 alert를 띄워 '서버에 문제가 있습니다.' 라고 한다.
